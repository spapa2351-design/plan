# LMS Vision 2026 — 핵심 요약

## 대전제
- **현재 온라인 코스는 레거시**. 미래에는 에디터 기반 인덱싱 + 학습 에셋 조합의 새 코스 구조로 대체됨
- **오프라인 코스가 Phase 1 출발점** — 차세대 UI/UX 뼈대를 오프라인에서 먼저 확립

## 1. 명칭 재정의: 페이지 → 학습 아이템(Asset)
- AS-IS: 커리큘럼 > 페이지(Page) — 이북처럼 껍데기 안에 종속
- TO-BE: 커리큘럼 > 학습 아이템(Asset) — 스포티파이 재생목록처럼 독립 자산을 참조(Link)

## 2. 커리큘럼 설계: 인덱싱 워크플로우
- **뼈대 먼저, 살은 나중에** — 관리자가 목차(Index/Slot)를 먼저 잡고, 라이브러리에서 에셋을 끌어와 참조 매핑
- 목차(Slot) = 커리큘럼의 골격, 에셋 = 교체 가능한 콘텐츠

## 3. 에셋 거버넌스 (생애주기)
| 상태 | 설명 |
|---|---|
| Draft (초안) | 수강생 없음. 무제한 수정 가능 |
| Published (활성) | 수강 데이터 축적 중. 수정 시 '안전한 교체 시나리오' 작동 |
| Archived (보관) | 미사용. 과거 통계 영구 보존 |

- 편집 자체를 막는 것은 불가능 → 핵심은 **수정의 파급효과를 운영자에게 경고**하는 것

## 4. 데이터 무결성: 슬롯(Slot) 중심 진도율 추적
- 진도율 기준 = 특정 영상 파일이 아닌 **커리큘럼의 목차(Slot)**
- Slot Pass → 에셋이 v1→v2로 교체되어도 기존 수강생 진도율 100% 보존
- 에셋 버전 추적 DB 불필요 → 개발 공수 극적 감소

## 5. 실행 로드맵
| Phase | 내용 |
|---|---|
| **Phase 1** | 🏢 오프라인 코스 선(先) 개발 — 차세대 UI/UX 뼈대 + 출결 모듈 확립 (← **현재 여기**) |
| **Phase 2** | 💻 온라인 코스 (가벼운 조립) — 무거운 저작 도구 탈피, 슬롯 기반 에셋 참조형 플레이리스트 |
| **Phase 3** | 🔄 블렌디드 코스 완성 — 오프라인 출석 + 온라인 콘텐츠를 하나의 커리큘럼 UI에서 융합 |

**Why:** 오프라인 코스 작업이 단순한 기능 추가가 아니라 차세대 LMS 아키텍처의 기반 작업임을 항상 인지해야 함

**How to apply:** 
- 코스 목록(index.html)은 온라인/오프라인/블렌디드 공존 구조이지만, 현재 온라인은 레거시 → 오프라인 중심으로 설계 우선
- "필수" 같은 레거시 개념보다 에셋/슬롯/참조 기반의 새 개념으로 사고
- 코스 빌더(admin1)의 패키지/회차 구조가 미래의 슬롯 기반 인덱싱의 시작점
