1배경과 정합성 원칙

현재 구조의 한계

현재 운영 중인 커리큘럼-페이지 구조는 학습자가 페이지에 진입하는 순간 진도율을 100%로 처리한다. 이 방식은 다음 문제를 안고 있다.

  • 진도율 숫자가 실제 학습 여부를 반영하지 못함
  • 수료율 통계가 클릭률에 가까움
  • 고용보험 환급 감사 시 학습 사실 증빙 불가

정합성 5원칙 — 모든 설계의 최상위 원칙

원칙 1
행위 기반 Pass
페이지 진입이 아닌 실제 학습 행위에만 Pass 처리
원칙 2
불변 기록
Pass는 수정·삭제 불가. 취소 시 별도 이벤트 추가
원칙 3
증거 동반
시청 시간·점수 등 "왜 Pass인지" 함께 기록
원칙 4
Slot 귀속
기록은 자료가 아닌 목차 자리에 귀속됨
원칙 5
집계=캐시
대시보드 숫자는 원시 이벤트로 재계산 가능

2학습 자료 체계

학습 자료는 두 계열로 나뉜다. 이 분류가 모든 수정 정책의 출발점이다.

📺 소비형 (Consumable) — 보기/읽기만 하는 자료

유형Pass 조건사고 위험
영상전체 러닝타임의 90% 이상 실질 재생낮음
글 (아티클)스크롤 100% 도달 + 최소 체류 시간낮음
PDF전체 페이지 열람 + 최소 체류 시간보통

✍️ 응답형 (Interactive) — 답을 남기는 자료

유형Pass 조건사고 위험
설문전 문항 응답 후 제출보통
퀴즈통과 점수 달성높음
시험수료 기준 통과매우 높음
💡 응답형 자료는 응답이 1건이라도 발생하는 순간 자동 잠금 상태가 된다.

3자료 유형별 수정 정책

📺 영상 · 글 (공통 2단 구조)

수정 유형적용 범위기존 학습자 영향
살짝 고치기오타, 로고, 이미지 등 내용 불변 수정진도율·이어보기 위치 모두 유지
크게 바꾸기강사 교체, 내용 개정 등 내용 변경Pass 유지. 재시청 알림 선택 가능

📄 PDF 다운로드 정책

옵션설명증빙권장 대상
A. 뷰어 전용다운로드 차단, 뷰어 내 열람만높음보안 민감 과정
B. 다운로드 허용편의성 우선, 뷰어 내 Pass만 기록낮음사내 일반 교육
C. 다운로드 + 퀴즈다운로드 허용, Pass는 확인 퀴즈로높음고용보험 환급 과정 (권장)

📋 설문

  • 응답 0건 상태: 자유 수정. 첫 응답 발생 시 자동 잠금
  • 뜻 불변 수정: 오탈자 등. 운영자 체크박스 선언 시 허용
  • 뜻 변경 수정: 시스템 차단. 새 버전(v2) 발행만 허용

🧩 퀴즈

  • 선지 오타: 뜻 불변 수정. 재채점 없음
  • 정답 오류: 전원 재채점. 합격 전환 정책 사전 확정 필수
  • 문제 교체: 기존 Pass 유지. 옛 문제 응답 스냅샷 영구 보존

🎓 시험 (가장 엄격)

  • 응시 시작 후 수정 시도 → 완전 잠금
  • 정답 긴급 수정 → 최고관리자 + 사유 + 2인 승인
  • 응답 보존 기간 → 최소 5년 (고용보험 환급 기준, 법무 확인 사항)

4콘텐츠 라이브러리 구조

콘텐츠 라이브러리는 회사가 보유한 모든 학습 자료의 단일 원본 저장소다. 자료는 라이브러리에 단 한 번만 저장되고, 여러 코스·슬롯에서 참조(Link)해서 사용한다.

업로드 경로 — 두 경로, 하나의 저장소

경로특징적합 상황
A. 라이브러리 선 업로드콘텐츠팀이 미리 쌓아두고 코스 기획 시 선택여러 코스에서 재사용할 자료
B. 코스 인덱싱 중 업로드슬롯 생성과 동시에 업로드 (자동 라이브러리 등록)해당 코스 전용 자료
💡 두 경로 모두 최종 저장소는 하나다. UX만 두 가지, 데이터 구조는 통일.

자료 상태 (Status)

상태정의수정 가능 여부
🟦 Draft업로드됐으나 어떤 슬롯에도 미연결자유 수정
🟩 Published1개 이상 슬롯에 연결, 학습자 노출 중유형별 정책 적용
⬜ Archived모든 슬롯 연결 해제됨. 과거 기록 보존용수정 불가
📌 상태 전환 규칙 — 첫 슬롯 연결 시 Draft→Published / 마지막 슬롯 해제 시 Published→Archived / Archived 자료는 영구 보존 (삭제 불가)

5데이터 구조 (개발팀 참고)

모든 핵심 테이블은 INSERT ONLY로 운영한다. 기록은 쌓이기만 하며 지워지지 않는다.

asset (라이브러리의 자료 원본) id / type (video | article | pdf | survey | quiz | exam) status (draft | published | archived) version_label uploaded_by / uploaded_at has_responses ← 응답형 자료의 Freeze 트리거 slot (커리큘럼의 목차 자리) id / course_id / order / slot_name slot_asset_binding (INSERT ONLY — 연결 이력) slot_id / asset_id / asset_version bound_at / unbound_at pass_event (INSERT ONLY — 통과 이벤트) user_id / slot_id asset_id + version pass_type / evidence (JSON) occurred_at response_snapshot (응답형 전용, INSERT ONLY) user_id / slot_id / asset_id + version question_snapshot (JSON) user_answer / scored_result asset_change_log (INSERT ONLY — 변경 이력) asset_id / changed_by / change_type before / after / reason_text approved_by (시험 정답 수정 시 필수)
API 공통 규칙
  • 응답형 자료는 has_responses = true 시 수정 API 423 Locked 응답
  • 예외 허용은 change_type=minor + 운영자 명시적 승인 토큰 필요
  • 시험 정답 수정은 최고 관리자 권한 + 2인 승인 워크플로
  • 모든 변경은 감사 로그에 영구 기록

6레거시 전환 전략

기존의 "페이지 열람 = 100%" 기록은 차세대 정합성 기준을 충족하지 못하지만, 학습자의 기존 수료 이력을 무효화하지 않는다. 명시적 라벨링 방식으로 공존시킨다.

전환 규칙

  • 기존 Pass 기록은 evidence_type = 'legacy_page_view'로 이관
  • 리포트에서 구버전 수료자와 신버전 수료자 구분 표시
  • 신규 오픈 과정부터 차세대 기준(행위 기반 Pass) 적용
  • 고객사 보고서에 두 기준 혼재 시 반드시 라벨 구분
💬 영업·교육 커뮤니케이션 — 고객사가 "이전 학습자 수료율이 왜 다르게 보이는가" 질문 시, 신구 기준 차이 설명 가능해야 함. 공통 설명 스크립트 별도 제공.

⚡ 경쟁 차별점

  • 슬롯 기반 진도율 보존 — 타 LMS는 자료 교체 시 진도율 초기화 사례 많음. 차세대 LMS는 목차(슬롯) 단위로 기록이 귀속되어 자료 교체와 무관하게 유지.
  • 응답 스냅샷 영구 보존 — 수년 전 시험 문항과 학습자 응답을 그대로 증빙 가능. 분쟁·감사 대응 강점.
  • 고용보험 환급 감사 대응 구조 내장 — 증빙 로직이 시스템 레벨에 설계되어 있어 고객사 별도 작업 불필요.
  • 부분 수정·즉시 반영 — 영상 오류 발견 시 전체 재배포 없이 해당 영상만 교체. 운영 부담 최소화.
역할 — 본 정책의 기술적 구현. 정합성 5원칙이 시스템 레벨에서 강제되도록 설계.

핵심 구현 원칙

  • 모든 Pass 기록은 이벤트 소싱 (UPDATE 금지, INSERT ONLY)
  • 응답형 자료의 Freeze 로직을 API 게이트웨이 레벨에서 강제
  • 감사 로그는 별도 저장소·별도 백업 주기 적용
  • 응답 스냅샷은 그 시점 문항·선지 전체를 JSON으로 저장
