Governance & Workflow

에셋 거버넌스: 편집 제한이 아닌 '안전한 관리'

"편집을 아예 못하게 하는 것은 불가능합니다." 핵심은 에셋의 생애주기(Status)를 정의하고, 학습 데이터가 쌓인 시점부터 **수정의 파급효과를 운영자에게 경고**하는 것입니다.

📝
초안 상태 (Draft)
수강생이 아직 없습니다. 에셋의 파일, 제목, 설정을 무제한으로 수정할 수 있습니다.
🚀
활성 상태 (Published)
수강 데이터가 쌓이기 시작했습니다. 수정 시 **'안전한 교체 시나리오'**가 작동하여 데이터 훼손을 막습니다.
📦
보관 상태 (Archived)
더 이상 사용하지 않지만, 과거의 수강 통계를 보존하기 위해 시스템 뒤편에 영구 저장됩니다.

Data Integrity

데이터 무결성 전략: '목차(Slot)' 중심의 진도율 추적

에셋(영상)이 변경될 때마다 버저닝(Versioning)을 추적해야 하는 복잡한 개발 구조를 탈피했습니다. 진도율 추적의 기준을 특정 영상 파일이 아닌 '커리큘럼의 목차(Slot)'로 상향 매핑하여 완벽한 데이터 무결성과 압도적인 개발 효율을 확보합니다.

✅
Slot(목차) 통과 기준 달성
학습자가 '1강' 목차에 할당된 영상을 끝까지 시청하면 시스템은 해당 영상의 버전에 관계없이 '1강 슬롯(Slot): Pass'라는 확정된 도장을 찍습니다.
🛡️
과거 학습 데이터 영구 보호
이후 관리자가 1강 영상을 v1에서 v2로 완전히 갈아 끼우더라도, 이미 '1강 Pass' 판정을 받은 기존 수강생의 진도율 100%는 절대 훼손되지 않고 그대로 보존됩니다.
📉
개발 & 운영 리소스 최소화
백엔드는 무거운 에셋 버전 추적 DB를 설계할 필요가 없어 개발 공수가 극적으로 감소합니다. 운영자 역시 통계 꼬임 걱정 없이 언제든 원하는 슬롯의 에셋만 쏙 빼서 최신화할 수 있습니다.

전략적 실행 로드맵 (Execution Plan)

안정적인 아키텍처 전환을 위해 3단계 실행 로드맵을 제시합니다.

Phase 1

🏢 오프라인 코스 선(先) 개발

기존 온라인 구조를 건드리지 않고, '현대적 UI/UX'의 뼈대와 출결 모듈을 안전하게 확립합니다.

Game Changer
Phase 2

💻 온라인 코스 (가벼운 조립)

무거운 저작 도구를 탈피하고, 슬롯(Slot) 기반의 가벼운 '에셋 참조형 플레이리스트' 세팅을 구현합니다.

Phase 3

🔄 블렌디드 코스 완성

독립 모듈로 완성된 오프라인 출석과 온라인 콘텐츠를 하나의 커리큘럼 UI에서 융합합니다.

🛠️

에셋 교체/수정 시뮬레이션

현재 에셋은 **12개 코스**에서 사용 중이며, **450명**의 누적 학습자가 존재합니다.
