# 오프라인 코스 시스템 (Offline Course Management)

## 파일 구성

| 파일 | 역할 | 경로 |
|---|---|---|
| `index.html` | 코스 목록 (통합: 온라인/오프라인/블렌디드) | `offline/index.html` |

# 1. 코스 목록 (index.html)

## 레이아웃
- 상단 헤더: "코스 ★" + 코스 순서 관리 / 수강생 등록 / + 등록 버튼
- 좌측 사이드바: 분류 트리 (WORK > 직무역량/리더십/영업, 공통/온보딩, IT, DX)
- 메인 영역: 필터 바 + 테이블

## 필터 체계
- **타입 탭**: 전체 | 오프라인 | 온라인 | 블렌디드 (각 카운트 표시)
- **상태 드롭다운**: 전체 / 운영중 / 임시저장 / 종료
- **검색**: 코스명 / 강사명
- **분류 사이드바**: 카테고리 트리 클릭 필터

## 테이블 컬럼 (온·오프 통합)
체크 | 확장▶ | 상태 | 필수 | 표지 | 코스명 | 구성 | 분류 | 수강기간 | 수강인원 | 등록일 | 생성자 | 관리

### 컬럼 적응형 렌더링 (브레인스토밍 결정)
| 컬럼 | 온라인 | 오프라인/블렌디드 |
|---|---|---|
| 코스명 | 제목 | 제목 + `오프라인` 뱃지 + "3개 반" + 장소 서브텍스트 |
| 구성 | `수강 관리` 링크 | `수강 관리` + `4회차` 버튼 |
| 수강기간 | 제한 없음 / - | 패키지 중 가장 이른~늦은 날짜 요약 |
| 수강인원 | 합산 | 패키지 합산 총원 |

### 확장 행 (Expandable Row)
오프라인/블렌디드 코스 행의 ▶ 클릭 → 인라인 패키지 미니 테이블 펼침
- 패키지명, 기간, 장소, 회차, 정원, 신청, **잔여석** (마감/임박 컬러)
- 하단: 패키지 편집 → admin1.html / 출석 관리 바로가기

### 행 단위 = "코스" (브레인스토밍 결정)
패키지 단위로 하면 1코스×5패키지=5행 → 목록 폭발. 코스 단위가 온라인과 일관성 유지.

# 2. 코스 개설/편집 빌더 (admin1.html)

## 레이아웃
- 상단 헤더: ← 목록 돌아가기 + 코스명 + 상태 뱃지(초안/임시저장/퍼블리싱됨) + 임시저장 버튼 + 미리보기 버튼
- 좌측 메인(800px): 섹션 1~3
- 우측 사이드바(500px): 커리큘럼 관리 패널
- 하단 고정: 코스 퍼블리싱 버튼 (사이드바 내)

## 섹션 1: 코스 기본 정보
**상태: `courseInfo`**
- title, category, instructor, instructorDesc, intro, location, address, traffic

**UI**: 코스명, 상단 표시 방식(기본표지/이미지), 카테고리, 강사, 강사소개, 코스소개, 장소설정(장소명/주소검색/교통안내)

## 섹션 2: 모집 및 수강 신청 설정

### 수강 신청 기간 (항상 고정, 섹션 최상단)
- **결정**: 오프라인 코스에 무제한 개념 없음 → 자율/강제 구분 없이 항상 표시
- 신청 오픈/마감 datetime-local

### 수강 등록 방식
1. **자율 수강** (`isSelfEnroll=true`)
   - 신청 권한 그룹 지정
   - 옵션 선택 규칙: 단일 선택형(택1) / 다중 조합형(최대 N개)
   - **다중 조합형**: 최소 제거, 최대만 유지 (브레인스토밍 결정)

2. **관리자 강제 할당** (`isSelfEnroll=false`)
   - 취소·변경 정책 UI 숨김 (관리자가 직접 처리하므로)
   - 패키지에 그룹 칩 배정 UI (정원 자동 계산)

### 취소·변경 정책 (자율 수강 전용)
**상태**: `cancelPolicy`, `changePolicy`

**수강 취소 정책** (토글 ON/OFF):
- OFF → 신청 마감일 전까지 언제든 취소 가능
- ON → 라디오 2택:
  - **기한부 취소 허용**: "교육 시작 N일/시간 전까지 가능" (기한 후 관리자 승인 필요)
  - **취소 불가**: 관리자만 취소 처리 가능

**일정 변경 제한** (토글 ON/OFF):
- OFF → 신청 마감일 전까지 옵션 변경 가능
- ON → "교육 시작 N일/시간 전까지 변경 가능" (amber 색상)

**결정**: 특정 일시 지정 방식 삭제 → 교육 시작 기준만 유지 (단순화)

## 섹션 3: 선택 옵션(패키지) 생성

### 데이터 모델: `options` 배열
```js
{
  id: number,
  name: string,          // 옵션명
  capacity: number,      // 정원 (그룹 있으면 자동계산)
  enrolled: number,      // 신청 인원
  startDate: string,     // 패키지 시작일
  endDate: string,       // 패키지 종료일
  place: string,         // 개별 장소 (비어있으면 기본 장소)
  groups: [{ id, name, count }],  // 강제 할당 시 배정 그룹
  sessions: [            // 세부 회차 배열
    { id, name, date, startTime, endTime, place }
  ]
}
```

### 패키지 카드 기능
- 인라인 이름 수정, 정원 입력 (그룹 있으면 자동 → 읽기전용 + "자동" 뱃지)
- 시작/종료 날짜, 개별 장소 필드
- 강제 할당 모드: 그룹 칩 UI (드롭다운으로 추가/X로 제거, 합산 표시)
- 복사, 삭제 버튼

## 우측 사이드바: 커리큘럼 관리

### 패키지 간 DnD
- `draggable` + GripVertical 핸들
- 드래그 중: drag-ghost (반투명), 드롭 대상: drag-over-opt (인디고 테두리)

### 패키지별 영역
- 헤더: 패키지명, N회차 뱃지, 일괄 셋팅 버튼, 접기/펴기 토글
- 회차 목록: 카드 형태, 패키지 내 DnD (같은 패키지 안에서만)
- 회차 카드: 회차명 input, 날짜, 시작/종료 시간, 출석QR 버튼, 삭제
- 회차 추가 / 일괄 셋팅 (기간+요일+시간 → 자동 생성)

### QR 출석 모달
- 회차별 QR 코드 (Google Charts API)
- 코스명, 옵션명, 날짜, 시간 인코딩

## 퍼블리싱 플로우

### 코스 상태: `courseStatus`
- `draft` → `saved` → `published`
- 상태 뱃지 (헤더에 색상별 표시)
- 마지막 저장 시각 표시

### 임시저장
- 언제든 클릭 가능, courseStatus → 'saved' 전환

### 퍼블리싱 모달 (코스 퍼블리싱 버튼 클릭 시)
1. **등록 요건 확인** — 7개 항목 자동 검증 체크리스트:
   - 코스명 입력, 담당 강사, 교육 장소, 신청 기간, 최소 1개 패키지, 모든 패키지에 회차 존재, 모든 회차 날짜/시간 설정
2. **공개 시점 선택** (모든 요건 충족 시만 노출):
   - 즉시 공개 / 예약 공개 (datetime picker)
3. **경고 안내**: 퍼블리싱 후 신청 시작, 수정 시 알림, 삭제 시 자동 취소
4. **푸터**: 취소 / 임시저장만 / 퍼블리싱 확정 (요건 미충족 시 비활성)

## 모바일 미리보기 (`isPreviewOpen`)
- 폰 프레임(340×720) 모달
- 코스 헤더: 신청 기간, 코스명, 강사, 장소, 교통
- **강제 할당 모드**: amber 관리자 배정 블록 + 패키지 목록(날짜/장소/그룹 칩)
- **단일 선택형**: 카드 리스트 (잔여석/마감 뱃지)
- **다중 조합형**: 체크박스 카드 + "최대 N개" 안내
- "교육 신청 확정" 버튼

# 3. 수강 관리 (admin1-1.html)

## 브레인스토밍 결정사항

### 탭 구조 (2026-04-09 재설계)
온라인: 요약|수강|동영상|퀴즈|시험|설문|의견|PDF|이미지
→ 오프라인: **요약 | 수강 | 출석 | 설문**
(동영상/퀴즈/시험/PDF/이미지/일정 제거, 출석 신설, 일정은 코스편집에서 관리)

### 출석 상태 5종 (2026-04-09 확정)
| 상태 | 키 | 발생 조건 | 최종 상태 여부 |
|---|---|---|---|
| 미확인 | `pending` | 회차 생성 시 전원 기본값. **임시 상태** | ✕ (반드시 다른 상태로 전환) |
| 출석 | `attended` | QR 스캔(시작 시간 내) 또는 관리자 수동 처리 | ○ |
| 지각 | `late` | QR 스캔(시작 시간 이후) 또는 관리자 판단 | ○ |
| 조퇴 | `early_leave` | 교육 도중 퇴장, 관리자 직접 변경 | ○ |
| 결석 | `absent` | 관리자 수동 또는 **회차 마감 시 미확인→결석 일괄 전환** | ○ |

### 출석인정(excused) 제거 이유
- 관리자가 수동으로 "출석" 처리하면 그게 곧 출석인정
- 6종→5종 단순화로 관리자 혼란 감소

### 미확인(pending) 정책
- **정의**: "아직 판단 전" 임시 상태, 최종 상태가 아님
- **라이프사이클**: 회차 생성(전원 미확인) → 교육 중(QR 스캔→출석/지각) → 교육 종료(관리자 개별 처리) → 회차 마감(미확인→결석 일괄)
- **미확인이 남는 케이스**: ①교육 시작 전, ②QR 안 찍은 수강생, ③QR 오류/폰 없음, ④관리자 확인 중
- **회차 마감**: 관리자가 "회차 마감" 버튼 클릭 → 확인 모달("미확인 N명이 결석으로 처리됩니다") → 일괄 전환. 마감 후에도 개별 상태 변경 가능
- **자동 마감 없음**: 관리자 수동 마감만. 이유: 자동 결석 시 "QR 못 찍었는데 결석됐어요" 민원 방지

### 출석 관리 UX (2026-04-09 결정)
- **미확인 상태**: [출석] 원클릭 버튼 + [...] 드롭다운(지각/결석) 병렬 제공
- **확정 상태**: 드롭다운으로 상태 변경 (출석/지각/조퇴/결석/미확인)
- 이전 "출석인정" 버튼 → "출석" 버튼으로 변경 (혼동 제거)

### QR 자동 갱신
- **30초 주기** (브레인스토밍 결정)
- 10초: 너무 짧아 스캔 실패 / 60초: 대리촬영 공유 위험
- QR에 타임스탬프+회차ID 포함 → 1회성 처리

### 관리자 TOP 5 액션 (우선순위)
1. QR 출석 화면 띄우기 (매 회차 시작)
2. 미출석자 확인 및 연락
3. 개별 상태 변경 (지각/조퇴/결석)
4. 회차별 출석 현황 요약 확인
5. 회차 마감 처리 (미확인→결석 일괄)

### 패키지 전환
하루 1~2개가 일반적 → 사이드바 드롭다운이면 충분

## 레이아웃
- 상단 헤더: ← 목록 + 코스명 + 브레드크럼 + 수강생 등록 버튼
- 좌측 사이드바(210px): 코스 표지, 코스명, **패키지 드롭다운**, 상태/정원/잔여석/회차, 교육 기간/장소, QR 코드(소형), 분류, 강사, 종료 버튼
- 메인 탭 영역

## 요약 탭
- 통계 카드 4개: 총인원, 출석인원(미출석 수), 평균출석률, 다음교육일
- 회차별 출석률 바 차트
- 패키지(반) 비교 테이블

## 수강 탭
- 수강생 테이블: 체크 | 상태 | 사진 | 이름 | 아이디 | 그룹 | 출석률 | 수강기간 | 상세 | 관리
- 상단 액션: 전체 수강생 삭제 / 전체 수강 기간 변경

## 출석 탭
- **회차 선택 바**: 패키지 내 회차 칩 버튼 (출석률 표시) + **라이브 출석 모드** 버튼
- 회차 정보 요약: 출석 기간, 출석 수, 미출석 수
- 출석 테이블: 체크 | 상태(6종 뱃지) | 사진 | 이름 | 아이디 | 그룹 | 출석시간 | 관리
  - 미확인 → "출석인정" 버튼
  - 출석/지각 등 → 상태 변경 드롭다운

## 라이브 출석 모드 (전체화면, 프로젝터/모니터용)
- **좌측 70%**: QR 코드 (대형) + 30초 카운트다운 + 자동갱신 표시
- **우측 30%**: 다크 대시보드
  - 실시간 출석자 수 (48px 대형 숫자) / 총원
  - 프로그레스 바 + 출석률%
  - 상태별 카운트 (출석/지각/출석인정/결석/미확인)
  - 최근 출석자 리스트 (실시간 추가 애니메이션)
  - 하단: 회차 전환 드롭다운
- 토스트: 출석 상태 변경 시 알림

## 일정 탭
- 회차별 타임라인 카드: 번호, 회차명, 날짜, 시간, 장소, 출석 현황, 완료 뱃지
- "코스 편집" → admin1.html 링크

# 메뉴 구조 결정 (2026-04-09 브레인스토밍)

## 핵심 결정: 기존 코스와 메뉴 분리
- 오프라인 코스 = 차세대 코스 시스템의 Phase 1이므로 기존 온라인 코스와 **별도 메뉴**로 분리
- 기존 온라인 코스는 레거시로 취급 (신규 개설 X, 유지보수만)

## GNB 메뉴명 로드맵
| Phase | 차세대 메뉴명 | 기존 메뉴명 |
|---|---|---|
| Phase 1 (현재) | **오프라인 코스** | 코스 관리 (기존 그대로) |
| Phase 2 | 코스 관리 (리네이밍) | 레거시 코스 (리네이밍) |
| Phase 3 | 코스 관리 (통합) | 레거시 코스 (아카이브 → 제거) |

## Phase 1 메뉴명이 "오프라인 코스"인 이유
- 실제로 오프라인밖에 없으니 이름과 실체 일치
- 기존 사용자 혼란 없음
- Phase 2에서 온라인 에셋 추가 시 자연스럽게 리네이밍

## index.html 영향
- 타입 탭(전체/오프라인/온라인/블렌디드) → Phase 1에서는 오프라인 전용이므로 재검토 필요
- 상태 필터(운영중/임시저장/종료)가 메인 필터로 승격 가능성

# 핵심 설계 원칙

1. **메뉴 분리**: 차세대(오프라인 코스)와 레거시(기존 코스 관리)를 GNB에서 분리
2. **확장 행(Expandable Row)**: 상세 정보는 인라인 펼침으로 해결
3. **관리자 현장 편의성**: 라이브 출석 모드, QR 30초 갱신, 큰 숫자 대시보드
4. **패키지 단위 관리**: 사이드바 드롭다운으로 빠른 전환, 요약 탭에서 전체 비교
5. **단일 HTML 파일**: 빌드 없이 CDN만으로 동작, 빠른 프로토타이핑
