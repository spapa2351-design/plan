## 1. 오프라인 코스 배지 정책 확정

이러닝 서비스 기획자와 상의하여 어드민 설정 정책과 1:1 매핑되는 6개 상태 배지 확정.

| 상태 | 배지 | 색상 | 조건 (어드민 정책 기준) |
|------|------|------|----------------------|
| 수강전(오픈코스) | ✔ 수강 신청 가능 | 흰 배경, 초록 텍스트 | 수강 신청 기간 내, 미신청 |
| 미시작 | 시작전 | 빨간 배경, 흰 텍스트 | 신청 완료, 출석 0회 |
| 수강중 | 수강중 | 파란 배경, 흰 텍스트 | 수강기간 내, 출석 1회 이상 |
| 수강예정 | 수강예정 | 회색 배경, 흰 텍스트 | 신청 완료, 수강기간 미시작 |
| 수강종료 | 수강종료 | 빨간 테두리, 빨간 텍스트 | 수강기간 만료, 수료 미충족 |
| 학습완료 | ✔ 학습 완료 | 흰 배경, 초록 텍스트 | 필수 조건(출석률, 과제 등) 모두 충족 |

**결정사항**: 오프라인 코스는 별도 영역에 배치하므로, 개별 카드에 "오프라인" 라벨 불필요.

## 2. 생성된 파일

| 파일 | 역할 |
|------|------|
| `index.html` | 모드 선택 (관리자/사용자) 진입점 |
| `admin-index.html` | 기존 어드민 페이지 (index.html에서 복사) |
| `home.html` | 사용자 홈 — 최근 수강 코스 + 오프라인 전용 영역 + 카테고리별 영역 |
| `category.html` | 카테고리 목록 — 코스/PLUS 코스/오프라인 코스 섹션, 필터 탭 |
| `course-list-badges.html` | 카테고리 상세 (3열 그리드) — 서브탭, 검색, 필터, 정렬 |
| `user-my.html` | MY 페이지 (기존, 수정) — 필터 라벨 변경 |
| `user1.html` | 코스 상세 (기존, 수정) — URL 파라미터 수강 상태 지원 |
| `user1-enrolled.html` | 수강 상세 v2 (기존, 수정) — 사이즈/위치 통일 |
| `user1-completed.html` | 완료 상세 (기존, 수정) — 사이즈/위치 통일 |

## 3. 네비게이션 구조

```
index.html (모드 선택)
├─ [관리자 모드] → admin-index.html
└─ [사용자 모드] → home.html
    ├─ 탭: 홈(active) / 카테고리 / MY
    ├─ 최근 수강 코스 (온+오프라인 혼합)
    │   └─ 수강중 카드 → user1.html?status=enrolled
    ├─ 오프라인 코스 영역
    │   ├─ 전체보기 → category.html?filter=offline
    │   └─ 카드 클릭 → user1.html (상태별 파라미터)
    └─ 직무교육 영역
        └─ 더보기 → category.html?filter=offline

category.html
├─ ← 뒤로가기 (history.back)
├─ 탭: 홈 / 카테고리(active) / MY
├─ 필터: 전체 / 온라인만 / 오프라인만
├─ ?filter=offline → 자동 오프라인 필터 적용
└─ 오프라인 카테고리 클릭 → course-list-badges.html

course-list-badges.html
├─ ← 뒤로가기 (history.back)
├─ 서브 카테고리 탭 (전체/프로젝트관리/데이터분석 등)
├─ 검색 + 필터 체크박스 + 정렬
└─ 3열 그리드 카드 → user1.html

user1.html (코스 상세)
├─ ?status=enrolled → 수강중 상태 자동 표시
├─ ?status=completed → 완료 상태 자동 표시
└─ "내 수강 상세 보기" → user1-enrolled.html

user-my.html (MY)
├─ 탭: 홈 / 카테고리 / MY(active)
├─ 필터: 전체 / 수강중 / 수강예정 / 수강종료
└─ 코스 카드 → user1-enrolled.html (v2 상세 직행)
```

**라우팅 규칙**: 홈/카테고리/코스목록에서 수강중 코스 클릭 → `user1.html?status=enrolled` (공용 코스상세). MY에서만 → `user1-enrolled.html` (v2 상세 직행).

## 4. MY 필터 ↔ 백엔드 상태 매핑 (확정)

| MY 필터 | 포함 상태 |
|---------|---------|
| 전체 | 모든 내 코스 |
| 수강중 | 시작전 + 수강중 (수강기간 내) |
| 수강예정 | 수강예정 (수강기간 미시작) |
| 수강종료 | 수강종료 + 학습완료 (기간 종료) |

## 5. UI/레이아웃 통일 사항

| 항목 | 값 |
|------|-----|
| 앱 가로 폭 | `width: 390px` (데스크톱), `max-width: 390px` (모바일) |
| 앱 세로 높이 | `height: 860px` (데스크톱 고정) |
| 수직 정렬 | `align-items: center` (화면 중앙) |
| border-radius | `var(--r-lg)` = 16px (전 페이지 통일) |
| 콘텐츠 좌우 여백 | 20px |

| 탭바 | `flex-shrink: 0` (앱 내부 하단, fixed 아님) |

## 6. 카테고리 페이지 구성

**섹션 (전체 필터 시)**:
- 코스: 라이브, 성과관리, 특별강의, 가이드, 메디컬 강의, 소통, 비즈니스 경영, 서비스 기획, 교육과정, 4차 산업혁명
- PLUS 코스: ESG강의, 교육과정, 어학, 자격증
- 오프라인 코스: 직무교육, 리더십, 법정의무교육, 워크숍, 신입사원, 외부연수

**필터 동작**: 전체 → 3섹션 모두 / 온라인만 → 코스+PLUS만 / 오프라인만 → 오프라인만

## 7. 코스 목록 (course-list-badges.html) 구성

- 헤더: ← + 카테고리명 (직무교육)
- 서브 카테고리 탭: 전체 / 프로젝트관리 / 데이터분석 / 커뮤니케이션 / 문제해결
- 검색바
- 필터: 수강신청가능(체크) / 수강중(체크) / 필수(토글)
- 건수 + 정렬: "12개 코스" + 추천순 + 그리드뷰 토글
- 3열 세로 스크롤 그리드 (카드: 커버 3:4 + 상단 배지 + 제목 + 하단 상태/진행률)
